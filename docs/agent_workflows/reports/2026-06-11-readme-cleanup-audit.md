## Decision: scripts/rebuild_sparse_vectors.py

**Kept.** Rationale:

- Works with server-mode Qdrant (uses `QdrantVectorStore` which supports both modes).
- Has proper error handling (checks `encoder.is_fitted()`, uses batched scroll/update).
- Serves a real maintenance need: rebuilding sparse vectors after vocabulary or encoder changes without re-chunking or re-embedding.
- Documented in both READMEs under the new "Maintenance Scripts" subsection.
- No test coverage added because the script is a thin orchestration wrapper over existing tested components (`QdrantVectorStore`, `SparseVectorEncoder`). Full integration testing would require a running Qdrant server with an existing collection.
